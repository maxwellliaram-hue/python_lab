 Worksheet Summary

I created a Python project folder named python_lab in my home directory and organized it into src, tests, and docs. Inside src, I created main.py, utils.py, and config.py. I used the terminal to create directories, files, and the initial project structure, and I verified the layout with a recursive directory listing.

I then initialized a Git repository in the project folder and created a .gitignore file to ignore __pycache__/ folders, .pyc files, and .env files. After staging and committing the initial files, I tested the Python program with multiple inputs to confirm that it correctly calculated the square, even/odd status, and Fahrenheit conversion. I also added a small test suite to verify the utility functions and committed those changes. Finally, I created a feature branch for the greeting enhancement and prepared the repository for GitHub publishing.

# Key points learned
- Organizing code into src, tests, and docs makes a project easier to manage and understand.
- .gitignore helps keep repositories clean by excluding temporary and sensitive files.
- Git commits record the history of a project and make it easier to track changes over time.
- Python modules can be connected using imports, allowing one file to use functions defined in another file.
